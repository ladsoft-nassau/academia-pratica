Git e Github. Para que serve e como usar?

Por: Thalys Rossi

Git : 

        Git é um programa de código aberto acessado por linha de comando, inicialmente criado para servir como um repositório de código e 
        controle de versionamento do projeto do kernel Linux. Seu principal contribuidor e idealizador é também o próprio criador
        do Linux, o engenheiro de software Linus Torvalds. O Git foi concebido após a empresa que desenvolvia a aplicação utilizada pelos 
        desenvolvedores do Linux para controle de versão retirou o suporte grátis extendido a eles. Desde 2005 o Git vemsendo utilizado 
        pelas mais variadas equipes de desenvolvimento ao redor do mundo para versionamento e como repositório facilmente acessível. 
        Ele se mostra útil tanto para projetos pessoais quanto para grandes projetos desenvolvidos em equipes, pois o Git cria uma 
        espécie de linha do tempo do código fonte do projeto, permitindo ver à cada modificação exatamente o que foi alterado dentro 
        do código. Esta funcionalidade permite uma soberania e autonomia tremenda sobre o projeto, pois caso alguma coisa vá mal 
        durante o desenvolvimento, sempre é possível voltar a um estado anterior.

Github: 

        Github é uma plataforma de hospedagem remota de repositórios Git. Ela serve para que desenvolvedores possam hospedar seus códigos na nuvem, 
        desta forma permitindo o acesso ao código fonte dos seus projetos a todos que abram seu repositório. Se mostra extremamente útil para 
        projetos de código aberto, ou como uma espécie de portfólio, que permite ao desenvolvedor exibir seus trabalhos. É bastante utilizado pela 
        comunidade de Open Source por permitir esse acesso a qualquer membro da comunidade que deseje contribuir com este projeto.
        A interface do Github é mais gráfica que a do Git, permitindo uma maior facilidade na hora de se utilizar suas funções.

Enfim, como se usa esse negócio?? 

        Para utilizar o Git é necessário instalá-lo em sua máquina em primeiro lugar (Chega aqui). 
        Após instalar a aplicação, você deve rodá-la através de linha de comando, podendo utilizar o terminal nativo de seu sistema operacional ou 
        um terminal integrado em um editor de texto como VS Studio Code que permite esta funcionalidade. O primeiríssimo comando que você deve passar 
        para o Git, caso já tenha sua pasta que será usada como repositório criada, deve ser o:
        git init Este comando inicializa um repositório na pasta que foi escolhida, sempre será utilizado ao inicializar um novo repositório local. 
        git add Este comando serve para enviar um arquivo para este repositório local, ATENÇÃO, ele ainda não foi salvo, para isso usamos o próximo comando.
        git commit Este comando é utilizado para salvar o ponto em que o código se encontra neste momento. Geralmente é passado com uma mensagem 
                   que é passada através do complemento -m "mensagem aqui"
        git log Serve para exibir todos os commits feitos em cima deste código.
        git status Exibe o estado dos commits que foram feitos até o momento.
        git show Apresenta uma modificação específica na linha do tempo do código.
        git branch Cria uma ramificação do commit sendo trabalhado, é usado para que alterações feitas neste código não impactem o código que está funcional até o momento. 
        git checkout Permite que o desenvolvedor transite entre as variações existentes no código.
        git merge Serve para unir duas ramificações, ou seja, ao criar uma branch do master, para enviar o código modificado para o master se usa este comando.
        git push Serve para enviar as alterações feitas localmente para o repositório remoto.
        git push Serve para solicitar alterações em um repositório criado por outro desenvolvedor.
        Tratei aqui de comandos básicos e introdutórios, mas que por sí já são extremamente poderosos e permitem aos desenvolvedores um maior controle sobre o que 
        está sendo feito em seu código.
