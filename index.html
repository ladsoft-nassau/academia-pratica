Git e Github. Para que serve e como usar?

Por: Gilberto Martins.

Git é um controlador de versão de códigos, ele funciona no seu local(pc), ele perimite manter um histórico de alteração deu sei arquivo, permitindo voltar na cersão anterior de antes de alguma alteração.
Github é como uma rede social de programadores/desemvolvedores, ela perimite postar o codigos e também perimite
clonar uma repositorio para fazer alguma alteração, e permite a colaboração de outros programadores.

O Git oferece:

  1. controle de história(permite ver todas as versões anterior do codigo ser alterado, e sinaliza onde e qual foi a alteração);
  2. permite trabalhar em equipe no mesmo;
  3. Ramificação do projeto(dividir o projeto em parte, e depois juntar);
  4. seguranção(niguem pode ver ou altera sua parte da ramificação);
  5. Organização.

Configurações do Git: Para configurar o nome e o e-mail do usuario:

  • para o nome: git config --global user.name"nome do usuario"
  • para o e-mail: git config --global user.email"e-mail do usuario"
  • para ver o nome e o e-mail que estão nas configurações do Git: git config user.name e git config user.name

    • Para clonar um repositorio:

  • git clone url no lugar da "url" deve coloca o endereço do repositorio no Github para que possa ser clonado.

    • O Git separa os arquivos em guatro estados bem definidos como os arquivos vão ser: untacked , unmodifiel, modifiel e staged

  • O untacked(não marcado): é o momento em que o qrquivo acabou de ser adicionando mais não foi visto pelo Git.
  • O unmodifiel(não modificado): é quando o arquivo acaba de ser visto pelo Git.
  • O modifiel(modificado): é quando o arquivo foi modificado e ainda não foi adicionando com essa alteração.
  • O staged(encenado): é quando o arquivo esta pronto para executar o commit, quando o commit for executado ele volta para o estagio de unmodifiel.

    • Para adicionar um arquivo no repositorio:

  • git add nome_arquivo

    • Para deletar um arquivo do repositorio:

  • git rm nome-arquivo

    • Para adicionar arquivos deletados:

  • git add --all
  • git add -A

    • Apos adicionar um arquivo deve ser feito o comando "commit" para que o Git possa criar o controle de versões desse aquivo. comando Commit:

  • git commit -m""

    • O commit deve sempre ter uma mensagem para que possa saber o que foi feito nessa versão do aquivo.

    Caso o arquivo esteja no estado modifiel é possivel ver qual foi a alteração:

  • git diff

    • Para saber quantos commits tem no repositorio e ver o histórico desse repositorio é so executar o comando log:

  • git log

    • Ele ira lista cada commit que foi executado nesse repositorio com o dia e a hora do commit e também ira mostra um
    conjuto de caracteris e numeros que são a chave de identificação do commit, se colocar a chave do commit de pois do log ira mostra a mensagem do commit:

  • git log chave-commit

    • Caso queira ver no histórico o nome dos arquivos que estão em cada commit:

  • git log --name-status

    • Criar um repositorio no Git Bash:

  • git add --bare

    • Para adicionar um repositorio:

  • git remote add nome-do-repositorio caminho-do-repositorio

    • Para sicronizar o repositorio com o servido:

  • git push nome-repositorio branch-que-estou-usando

    • Para puxar os dados do repositorio no servidor:

  • git pull nome-do-repositorio branch-que-estou-usando

    • Para usar um repositorio do GitHub:

  • git remote set-url nome-do-repositorio url-do-repositorio

    • Agora se for executado os comandos:

  • git push nome-do-repositorio branch-que-estou-usando
  • git pull nome-do-repositorio branch-que-estou-usando

    • Ira executar o push e o pull para o GitHub
