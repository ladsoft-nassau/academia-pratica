Git :

Git é um sistema de controle de versão, que foi criado com o objetivo de controlar o código fonte, o desenvolvimento e alteração que o mesmo vem sofrendo. É extremamente importante para se ter um controle do que foi feito no projeto, pois todas as alterações nele, ficam salvas dentro de um repositório, desde a mais antiga para a mais atual. Além de ser uma prática que vem sido adotada ao longo de anos.

Github:

Uma plataforma de hospedagem de código fonte com controle de versão, onde se hospeda o Git na web.

Iniciando um repositório pelo prompt de comando:

mkdir ladsoft-git

cd ladsoft-git

git init

Verificando o status com git status:

git status

Adicionando arquivos ao Git:

git add REAME.md

Commitando no Git:

observação: commitar alterações significa salvar o estado em q o codigo fonte está naquele momento, evitando que as implementações anteriores sejam perdidas

git commit -m "Primeiro Commit"

Enviando um repositório Git existente para o Github a partir da linha de comando

git remote add origin https://github.com/adriellyscsantos/modulogit.git

git push -u origin master

Clonando repositórios no Github:

git clone -url-do-repositório-que-deseja-clonar-

Solicitar mudanças de outros repositórios

º No Github, no projeto clonado no seu perfil, clicar na opção "New Pull Request"

º Clicar em "Create Pull Request", e argumentar sobre as modificações.

º Aguardar que as modificações sejam aceitas pelo proprietário.
