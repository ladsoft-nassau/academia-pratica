Git e Github. Para que serve e como usar?

Por: ISABELLE FIDELES

Git é uma metodologia de versionamento de código, considerando o git uma linha do tempo, onde você tem o poder de mudar essa linha temporal, seja voltando para versões anteriores ou até mesmo criar linhas paralelas, que nesse caso chamamos essa linha paralela de branch. Bom... o versionamento veio para economizar tempo e estresse, pois, se um grupo de pessoas estão alterando o mesmo projeto, no final, o versionamento fará seu papel de juntar todas as partes do projeto, quais foram alteradas ou adicionadas, por exemplo. Agora falando dos comandos básicos quais usei no power shell, você pode navegar entre as pastas diretamente do CMD ou ir normalmente nas pastas, que é o que faço, apertando o shift do teclado + botão direito, aparece a opção de usar o power shell ou CMD, dito isso, os comandos digitados no terminal vão fazer as operações naquela pasta, quando criei uma pasta, e abri o power shell, digitei "git clone" + o link do repósitório e assim, clonei na minha pasta o repositório em questão, mas quando trata-se de usar tal pasta como um novo repositório vazio, basta digitar o comando "git init", para ver se realmente foi criado, basta olhar na pasta que você está alterando e mandar exibir os arquivos ocultos, assim você verá um arquivo .git, onde dentro dele há outros arquivos. Falando um pouco mais da branch, como eu disse, pode ser considerado como linhas paralelas de uma principal, que o nome dessa principal é master, branch master. No commit existe uma parte para deixar um comentário, que é sempre importante escrever algo, breve, a respeito do que foi commitado. O README.md é importante para adicionar informações descritivas. O git status serve para varrer a pastar e verificar o que foi adicionada, deletado ou algo do tipo, é básicamente das um status da branch que você se encontra Os status que um arquivo pode ter é:
-Não trackeado = quando o repositório git sabe que ele existe
-Trackeado mas não commitado = qunado o repositório sabe que ele existe mas não está pronto pra ser enviado
-Commitado = quando o repositório sabe que ele existe e está pronto para ser enviado
-Nada para commitar
O git add . adicionar tudo da pasta no repositório, já o git commit -m "mensagem", registra os arquivos no git, E o git log mostra o histórico de tudo que foi feito.
Após alterar o projeto, é necessário da o git add . e depois verificar o status para ver se está tudo pronto para ser commitado, após commitar já está tudo no repositório local esperando dar o pull request
